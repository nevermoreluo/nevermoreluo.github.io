Haproxy

  • haproxy


Haproxy日志系统配置，以及限速简介

日志

haproxy默认是不配置日志的
因此需要日志的话需要自己配置

首先确保haproxy内的配置文件一般为/etc/haproxy/haproxy.cfg, 存在关于log的配置记住local1 notice只要与后面配置的rsysylog一致即可

修改rsyslog配置vim /etc/rsyslog.d/haproxy.conf内填入

重启rsyslog和haproxy即可
sudo systemctl restart rsyslog, sudo systemctl restart haproxy

haproxy针对每个user的限制

修改haproxy配置vim /etc/haproxy/haproxy.cfg

原理是基于用户每次流量的字节吞吐量进行统计10s内过期如果超过一定量则将该用户的请求全部转向延时服务器给予响应
该方法也有一个显著的缺点即用户开启下载时直至用户下载完成字节吞吐量才会一次性计入统计即下载时完全无限制直到下载完成时才计入统计
因此haproxy还是比较适合做用户访问频率限制而流量限制交给squid比较合适
访问频率限制其实也很简单将bytes_out_rate的统计改为http_req_rate统计修改限制参数即可此处不做延伸
详情可参阅文档

另外可以查看存储的键值表内的信息使用命令sudo watch -n 1 'echo "show table http_3333" | socat unix:/run/haproxy/admin.sock -' 即可实时查看http_3333的frontend中存储的表信息进行调试了
